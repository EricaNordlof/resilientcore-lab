# Lab scope and limitations

ResilientCore Lab is an independent learning and portfolio project.

## What is implemented in the repository

- provider-style PE/P/CE topology definition
- FRRouting-style reference configurations for an IS-IS underlay
- MP-BGP/VRF reference configuration concepts on PE nodes
- sample network-state telemetry and automated validation
- Ansible examples for pre-change evidence and operational checks
- change, rollback, security and third-line troubleshooting documentation
- static NOC-style dashboard

## What is intentionally presented as design/reference material

Advanced service-provider features such as production-grade SR-MPLS, LDP, SRv6, EVPN, VPWS and VPLS depend on platform capabilities, kernel dataplane support, vendor implementation and lab environment. They are documented here to demonstrate protocol understanding and design reasoning, not to claim that this public repository reproduces a production operator network.

## Professional positioning

The project should never be presented as seven years of carrier-network experience, Cisco production ownership, CCNA/CCNP certification, or security-cleared critical-infrastructure work. It is evidence of structured learning, technical documentation, automation mindset and the ability to turn requirements into a testable lab design.
